# Discrete Mathematics Lab 2: RSA Chat

Цей проект реалізує простий чат із шифруванням RSA та перевіркою цілісності повідомлень за допомогою хешування.

## Інструкції до запуску

Для роботи чату необхідно мати встановлений Python 3.6 або новішої версії.

1. **Запуск сервера**:
   Відкрийте термінал та запустіть файл `server.py`. Сервер почне прослуховувати порт `9001` на локальній адресі.
   ```bash
   python3 server.py
   ```

2. **Запуск клієнта**:
   Відкрийте новий термінал (або декілька для різних користувачів) та запустіть `client.py`.
   ```bash
   python3 client.py
   ```
   *Примітка: за замовчуванням у коді `client.py` прописаний нікнейм `b_g`. Ви можете змінити його безпосередньо у файлі в секції `if __name__ == "__main__"`.*

3. **Користування**:
   Введіть повідомлення в терміналі клієнта та натисніть Enter. Воно буде зашифроване, підписане хешем та надіслане через сервер усім іншим підключеним клієнтам.

## Коротке пояснення імплементації

Проект складається з трьох основних модулів:

- **Генерація простих чисел (`random_prime.py`)**: 
  Використовується алгоритм **Міллера-Рабіна** для перевірки чисел на простоту. Це дозволяє генерувати надійні 256-бітні прості числа, які є основою для створення ключів RSA.
  
- **Криптографія (RSA)**:
  Кожна сторона (сервер та клієнт) генерує власну пару ключів:
  - **Public Key (e, n)**: використовується для шифрування повідомлень, що надсилаються цій стороні.
  - **Private Key (d, n)**: використовується для розшифрування отриманих даних.
  Шифрування виконується посимвольно: кожен символ перетворюється на число (його код ASCII) та підноситься до степеня `e` за модулем `n`.

- **Цілісність даних (Hashing)**:
  Для захисту від випадкових спотворень даних використовується хеш-функція **djb2** (модифікована). Перед шифруванням клієнт обчислює хеш повідомлення. Отримувач після дешифрування повторно обчислює хеш та порівнює його з отриманим. Якщо вони не збігаються, програма виведе попередження про порушення цілісності.

- **Мережева взаємодія**:
  Використовуються стандартні TCP сокети. Сервер обробляє кожного клієнта в окремому потоці (`threading`), що забезпечує асинхронний обмін повідомленнями в реальному часі.
